# Кейс 09: Автоматическая модерация карточек товаров (Webbee ML)

## Краткое описание

**Работодатель:** Webbee ML — аутсорс ML-решений для e-commerce.

**Клиент:** крупный селлер на маркетплейсе (WB/Ozon) с большим ассортиментом — тысячи SKU, постоянный поток новых карточек от внутренних контент-менеджеров и поставщиков.

**Задача:** до публикации карточки на маркетплейсе автоматически проверять **фото** (запрещённый контент, чужие логотипы, качество), **текст** (спам, контакты вне платформы, некорректные описания) и **соответствие фото описанию**, выдавая вердикт `approve / reject / to_review` с причинами.

**Итог одной строкой:** многоголовая ML-система — CV-детекторы запрещённого контента (YOLO-family), **CLIP-подобная кросс-валидация фото ↔ текст**, NLP-классификатор и словари для проверки описаний, **LightGBM-агрегатор** всех скоров в финальный вердикт. Recall-фокус на опасных проверках, консервативный fallback в `to_review`. Работает в **pre-publish** режиме внутри контура клиента, снижает ручную нагрузку модераторов и долю отклонений маркетплейсом.
