# Команда и роли

**Состав:** 2 инженера со стороны Webbee ML + модераторы клиента как заказчики и источник разметки.

## Состав

| Роль | Сторона | Зона ответственности |
|---|---|---|
| **Data Scientist (Артур)** | Webbee ML | Вся ML-часть: CV-головы, NLP-голова, CLIP-like мультимодальная проверка, LightGBM-агрегатор, работа с разметкой, калибровка порогов, метрики |
| **Backend-инженер** | Webbee ML | Сервис модерации на FastAPI, интеграция с внутренней админкой клиента, интеграция с WB/Ozon API для публикации, очереди `to_review` / `reject`, хранение артефактов вердиктов, Docker-деплой, мониторинг в продакшне |
| **Модераторы клиента** (лид + команда) | Клиент | Заказчики; источник исторической разметки (их прошлые вердикты); текущие пользователи системы, которые обрабатывают очередь `to_review`; дают обратную связь по ошибкам модели |

**PM со стороны клиента** присутствовал на воркшопах и приёмках, но в явной роли в повседневной работе не участвовал — основная коммуникация шла через лид-модератора клиента.

## Моя зона ответственности (Артур, Data Scientist)

### Данные и разметка
- Собрал **обучающую выборку** из логов внутренних модераторов клиента (карточка → вердикт + причина).
- Провёл **анализ inter-annotator agreement** на пересечениях (карточки, которые видели несколько модераторов): где сходятся — золотой датасет, где расходятся — отдельная «серая зона».
- Выявил **систематические смещения** отдельных модераторов (кто-то строже по тексту, кто-то по фото) и перекалибровал лейблы.
- Провёл **воркшоп с лид-модератором клиента** — формализовали спорные случаи в задокументированный гайдлайн, который стал источником правды для модели и для новых сотрудников модерации.

### CV-головы
- **Дообучил YOLO-детектор** чужих логотипов и водяных знаков на открытых датасетах + на истории отклонений клиента + на **аугментациях** (искусственное наложение водяных знаков конкурентов на чистые фото).
- Подобрал **NSFW-модель** из opensource и настроил пороги под recall.
- Реализовал **детектор качества фото** (блюр через Лапласиан + небольшой CNN на остальное).

### Мультимодальная голова
- Выбрал подходящую **CLIP-подобную модель** и провёл **доменный fine-tune** на парах `фото ↔ название + ключевые атрибуты` из истории одобренных карточек клиента.
- **Калибровал пороги cosine similarity по категориям** (у одежды нормальный уровень похожести отличается от электроники).

### NLP-голова
- Обучил **классификатор текста** на sentence-transformer'е / BERT на разметке модераторов (классы: спам, контакт в тексте, нерелевантное описание, противоречие атрибутам).
- Собрал и расширял **словари** запрещённых фраз и «творческих» обходов контактов.
- Реализовал отдельную логику **проверки соответствия описания атрибутам карточки**.

### Агрегатор и калибровка
- Спроектировал набор **фичей агрегатора** (скоры и флаги всех голов + метаданные карточки).
- Обучил **LightGBM-агрегатор** с `class_weight` под recall-фокус.
- Подобрал **пороги** для разделения на `approve / reject / to_review`, явно включил **консервативный fallback** (если опасная голова срабатывает — принудительно `to_review` независимо от агрегатора).
- Реализовал **explanations**: по топ-факторам срабатывания генерируется читаемый текст причины для UI модератора и контент-менеджера.

### Валидация и мониторинг
- Rolling-evaluation модели на свежих вердиктах модераторов — чтобы ловить drift в содержимом карточек и поведении поставщиков.
- Настроил регулярный **retrain** и обновление словарей.
- Сделал дашборд production-метрик (доля `approve/reject/to_review`, средний recall по recall-critical классам, ошибки агрегатора).

## Взаимодействие

- **С backend-инженером Webbee** — плотно на всех этапах: контракты API сервиса, формат артефактов моделей, параллельный запуск голов, очереди, деплой, отладка ошибок в проде.
- **С лид-модератором клиента** — регулярно: воркшопы по спорным кейсам, формализация гайдлайна, демо, ручная проверка ошибок модели.
- **С модераторами клиента** (через лида) — обратная связь на ежедневной основе: что модель неправильно отклонила, где пропустила плохое, на каких категориях больше всего override'ов.
