# Команда и роли

**Состав:** 3 инженера + HR-отдел как заказчик.

## Состав

| Роль | Зона ответственности |
|---|---|
| **Data Scientist (Артур)** | Вся ML-часть: дообучение эмбеддера, Qdrant-индекс, BM25-ветка, промптинг и интеграция Qwen 72B через SGLang, offline-оценка (NDCG@k / MAP@k) |
| **Backend** | Сервис-оркестратор на FastAPI, интеграция с hh.ru API (кэш, ретраи, rate limits), объединение BM25+dense-кандидатов (RRF), вызовы SGLang, хранение откликов и фидбэка HR |
| **Frontend** | Веб-интерфейс для HR: список откликов с score и объяснениями от reranker'а, фильтры, кнопки фидбэка, экспорт шорт-листа |
| **HR-менеджеры** (заказчик) | Формализация структурированного профиля вакансии, приёмка, источник разметки, обратная связь по качеству выдачи |

## Моя зона ответственности (Артур, DS)

- **Собрал обучающую выборку** для эмбеддера: исторические наймы (позитивные пары вакансия↔резюме) + синтетические пары от self-hosted LLM + hard-negative mining.
- **Дообучил русскоязычный sentence-transformer** под HR-домен через contrastive-лосс.
- **Настроил Qdrant:** схема коллекции, payload'ы, параметры ANN-индекса; реализовал пайплайн индексации резюме.
- **Реализовал BM25-ветку** на `rank_bm25`, подобрал токенизацию и формат запроса из структурированного профиля вакансии.
- **Сделал объединение** BM25 и dense через RRF и подобрал top-K для передачи в reranker (по компромиссу качество/latency).
- **Настроил LLM-reranker:** развернул Qwen 72B на SGLang, написал и итеративно улучшал промпт для пары «вакансия ↔ резюме», задал формат ответа (score + короткое объяснение), настроил батчинг.
- **Оценил качество** всего пайплайна через NDCG@k / MAP@k на hold-out'е из исторических наймов + размеченных HR примеров.
- **Защищал решения** перед HR-командой (почему гибрид, почему LLM-reranker, почему именно 72B).

## Взаимодействие

- **С Backend** — плотно: согласование контрактов (формат структурированного профиля, формат ответа reranker'а), оптимизация end-to-end latency, отладка ошибок hh.ru API.
- **С Frontend** — эпизодически: как показывать score и объяснение, какие фильтры и поля нужны HR.
- **С HR** — на старте (формализация профиля вакансии и критериев релевантности) и регулярно при приёмке (демо на реальных вакансиях, сбор фидбэка).
