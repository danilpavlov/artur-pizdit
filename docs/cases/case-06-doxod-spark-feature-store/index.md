# Кейс 06 (мини): Витрина клиентских фичей на PySpark — ИК «ДОХОДЪ»

!!! info "Мини-кейс"
    PySpark / Big Data. Компактная версия: 3 страницы — обзор, флоу, детали.

## Краткое описание

**Домен:** данные и инфраструктура инвестиционной компании «ДОХОДЪ».

**Задача:** построить единый **feature store** — ночную витрину клиентских фичей, которую потребляют все ML-модели (сегментация, NBA, churn), чтобы фичи на обучение и инференс считались одинаково, воспроизводимо и «на момент». Старый pandas-ETL не справлялся с объёмами DWH и давал расхождения между витринами.

**Итог одной строкой:** **PySpark-джоба** на кластере, которая раз в сутки пересчитывает все клиентские фичи из транзакций, портфеля, KYC и истории коммуникаций; партиционированные **Parquet**-таблицы по дате и клиенту; оркестрация через **Airflow**; покрытие тестами.

## Связь с другими кейсами

- **Кейс 05 (Churn)** — читает те же фичи.

## Формулировка для резюме

> **Спроектировал и реализовал** PySpark-джобу для построения единого **feature store** клиентских фичей в инвесткомпании. Джоба читает из DWH транзакции, портфельные срезы, KYC и историю коммуникаций, считает десятки фичей по временным окнам, пишет партиционированный **Parquet**. Оркестрация через **Airflow**. Оптимизировал **джоины, партиционирование и шаффлы**, покрыл трансформации юнит-тестами. Витрина стала **единым источником фичей** для моделей сегментации, NBA и churn.
