# Контекст

## Компания

- **Название:** Инвестиционная компания «ДОХОДЪ».
- **Индустрия:** брокерское обслуживание, доверительное управление, инвестпродукты для физлиц.
- **Инфраструктура:** **on-prem без доступа в интернет** — жёсткое ограничение на стек и способ поставки зависимостей.

## Бизнес-задача

Компания взаимодействует с рядом внешних подрядчиков (вендоры, сервисные интеграторы, контрагенты по документообороту). Для работы им нужно передавать документы — договоры, отчёты, приложения, — но документы содержат **персональные данные клиентов**, которые по 152-ФЗ и внутренним политикам нельзя отдавать наружу в исходном виде.

До проекта маскирование делалось **вручную**: сотрудники проходили по документам, искали ПДн, замазывали их. Медленно, дорого, ошибкоопасно, не масштабируется.

Задача проекта — **автоматизировать этот процесс**: получить сервис, который на вход принимает исходный документ, а на выход отдаёт его визуальную копию с маскированными ПДн, пригодную для передачи подрядчикам.

## Какие сущности маскируем

- **ФИО клиентов** — имена, отчества, фамилии физлиц.
- **Документные идентификаторы** — серии и номера паспортов, ИНН, СНИЛС.
- **Финансовые идентификаторы** — номера брокерских счетов и договоров, суммы сделок.
- **Контакты и адреса** — телефоны, email, адреса проживания.

## Почему ML, а не regex

Часть сущностей (ИНН, СНИЛС, серии/номера паспортов, email, телефоны) хорошо ловится регулярками. Но **ФИО, адреса и контекстно-зависимые сущности** (например, «счёт №…» vs «расчётный счёт подрядчика №…» — первое маскируем, второе нет) регуляркой не закрываются. Нужна модель, которая понимает контекст.

## Ограничения

- **On-prem, без интернета.** Все зависимости, веса моделей, датасеты — ставятся офлайн, переносятся артефактами. Никаких `huggingface_hub.download`, `pip install` напрямую из сети.
- **Лицензионные.** Использовать можно только **opensource-библиотеки, совместимые с коммерческим использованием**. В частности, **PyMuPDF под AGPL — исключён**, потому что требует либо open-source всего продукта, либо покупки коммерческой лицензии у Artifex.
- **Комплаенс / ПДн.** Реальные клиентские документы **нельзя выносить** ни разметчикам, ни в публичные облака — ни для разметки, ни для обучения. Это сразу убивает классический «размечаем в Label Studio» подход.
- **Recall критичен.** Пропуск ПДн = утечка и нарушение; ложное срабатывание (замаскировали лишнее) = косметический дефект. Поэтому модель тюнилась и валидировалась в первую очередь по recall.
