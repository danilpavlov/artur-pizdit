# Команда и роли

**Всего в проекте:** 3 человека — небольшая кросс-функциональная команда.

## Состав

| Роль | Человек | Зона ответственности |
|---|---|---|
| **Data Scientist** | Артур | Модель: дообучение GliNER, генерация обучающей синтетики через self-hosted LLM, валидация, подбор порогов под recall-ориентированную метрику |
| **Python Backend** | коллега | Сервис на FastAPI, PDF-пайплайн (pdfplumber → reportlab → pypdf), интеграция OCR-ветки (Tesseract + pdf2image + Pillow), Docker, деплой on-prem |
| **System Analyst** | коллега | Требования от бизнеса и комплаенса, формализация списка сущностей для маскирования, приёмка, коммуникация с подрядчиками и внутренними стейкхолдерами |

## Моя зона ответственности (Артур, DS)

- Собрал обучающий датасет через **синтетику**: отобрал шаблоны типовых документов, настроил генерацию через self-hosted LLM с подстановкой алгоритмически сгенерированных ПДн-значений (чтобы ground-truth метки были точны).
- Подмешал **открытые русскоязычные NER-датасеты** (Nerel, Collection3 и т.п.) для устойчивости к общим именным сущностям.
- **Дообучил GliNER** через HuggingFace Transformers + PyTorch, вёл эксперименты, сравнивал конфигурации.
- Выбрал **Recall как главную метрику** (и обосновал это бизнесу: пропуск ПДн ≫ ложное срабатывание) и **настраивал пороги** модели под этот приоритет.
- Валидировал модель на hold-out из синтетики + небольшом наборе внутренних документов, размеченных руками внутри контура.
- **Передал бэкенду** обученную модель в виде артефакта (веса + конфиг + минимальный wrapper для инференса), согласовал контракт входов-выходов NER-шага.
- Участвовал в отладке end-to-end пайплайна на стороне бэкенда — в частности, в маппинге char offset'ов от GliNER обратно в bbox слов от pdfplumber/Tesseract.

## Взаимодействие

- **С Python Backend** — плотно, почти ежедневно: согласование контрактов модели, помощь с интеграцией, отладка граничных случаев в маппинге спанов.
- **С System Analyst** — на этапах старта проекта (согласование списка сущностей и критериев качества) и приёмки (демо на реальных документах, принятие метрик).
