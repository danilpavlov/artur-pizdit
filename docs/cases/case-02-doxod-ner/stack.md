# Технологический стек

Ключевое ограничение: **on-prem без интернета + только opensource-лицензии, совместимые с коммерческим использованием**. Это определило весь выбор инструментов.

## Основной стек

| Инструмент | Лицензия | Назначение | Почему именно он |
|---|---|---|---|
| Python 3.x | PSF | Основной язык | Стандарт индустрии, экосистема NLP и PDF |
| **GliNER** (fine-tuned) | Apache 2.0 | NER-модель | Гибкий zero/few-shot NER на трансформере, хорошо дообучается на русском, компактный инференс, совместим с on-prem |
| HuggingFace Transformers | Apache 2.0 | Обучение и инференс модели | Стандартный фреймворк для fine-tune трансформеров |
| PyTorch | BSD | ML-фреймворк | Бэкенд для Transformers, офлайн-поставка wheel'ов |
| **pdfplumber** | MIT | Парсинг текстовых PDF с bbox | Чистый Python, MIT, без AGPL, извлекает слова с координатами — этого достаточно для маппинга спанов |
| **reportlab** | BSD | Генерация overlay-PDF с redaction-прямоугольниками | Стандартная opensource-библиотека для программной генерации PDF |
| **pypdf** | BSD | Слияние overlay-страниц с исходным PDF | Лёгкий opensource-форк PyPDF2, BSD, без AGPL |
| **Tesseract** + pytesseract | Apache 2.0 | OCR для сканов | Индустриальный opensource-OCR, ставится из deb-пакета офлайн, поддерживает русский |
| pdf2image | MIT | Рендер страниц PDF в изображения (для OCR) | Тонкая обёртка над poppler, MIT |
| Pillow | HPND | Препроцессинг изображений (deskew, бинаризация) | Стандарт для обработки картинок в Python |
| python-docx | MIT | Чтение и модификация DOCX | Единственный вменяемый opensource-инструмент для Word |
| **FastAPI** | MIT | REST-сервис маскирования | Быстрый, асинхронный, автогенерируемый OpenAPI — удобно отдавать подрядчикам |
| Docker | Apache 2.0 | Упаковка и деплой | Стандарт упаковки для on-prem |
| Self-hosted LLM | — | Генерация синтетики для обучающего датасета | Нельзя звать облачные LLM с ПДн-контекстом → развернули LLM внутри контура |

## Почему НЕ PyMuPDF

PyMuPDF — это «очевидный» первый выбор для PDF-редактирования: у него есть нативный **redaction API** (`page.add_redact_annot` + `page.apply_redactions()`), который реально вырезает текст из PDF, а не просто рисует поверх.

**Но:** PyMuPDF распространяется под **AGPL-3.0**. Для коммерческого проприетарного продукта это означает либо:

1. Открывать исходники всего продукта под AGPL (неприемлемо),
2. Либо покупать **коммерческую лицензию** у Artifex Software (дорого и требует процедуры согласования).

Вместо этого собрали redaction-пайплайн вручную из BSD/MIT-совместимых кусочков: **pdfplumber** (извлечение bbox) → **reportlab** (рисование overlay-прямоугольников) → **pypdf** (слияние). Получилось чуть больше кода, но без лицензионных рисков.

## Почему НЕ облачные LLM / API

- Ни для разметки, ни для синтетики, ни для инференса нельзя было использовать внешние API — **контур без интернета** и **работа с ПДн** делают это невозможным физически и юридически.
- Self-hosted LLM внутри контура закрыл потребность в генерации вариативной синтетики.

## Почему НЕ Label Studio с ручной разметкой реальных документов

- Реальные документы содержат ПДн → выносить их аннотаторам нельзя.
- Решение через **синтетику + открытые датасеты** — обходит эту проблему и даёт точные ground-truth метки.
