# Достижения

## Бизнес-результаты

- **Ручной процесс маскирования заменён автоматическим.** Раньше сотрудники вычитывали каждый документ и замазывали ПДн руками; теперь документ за секунды проходит через сервис и возвращается готовым.
- **Разблокированы интеграции с подрядчиками.** Появилась техническая возможность безопасно отдавать документы наружу — раньше это упиралось в скорость ручного маскирования.
- **Сотни документов в день** стабильно прогоняются через сервис.
- **Снижение комплаенс-риска** по работе с ПДн: маскирование стало воспроизводимым и одинаковым для всех документов, пропал человеческий фактор.

## Технические результаты

- **Recall: 0.90–0.95** по ключевым типам сущностей (главная метрика — нельзя пропускать ПДн).
- **F1 ≈ 0.90** по сущностям в entity-level оценке.
- **Дообучение** на ~10–30 тыс. синтетических документов + открытые русскоязычные NER-датасеты.
- **Собственный PDF-редакт-пайплайн** на чистом opensource-стеке (pdfplumber + reportlab + pypdf), полностью совместимый с on-prem без интернета и без AGPL-зависимостей.
- **Единообразная обработка** трёх форматов входа: текстовый PDF, отсканированный PDF, DOCX.

## Формулировки для резюме

> **Разработал** NER-модель для маскирования персональных данных в документах инвестиционной компании. Дообучил **GliNER** на синтетическом датасете (~10–30 тыс. документов), сгенерированном через self-hosted LLM, с добавлением открытых русскоязычных NER-корпусов. Достиг **Recall 0.90–0.95** и **F1 ≈ 0.90** по сущностям (ФИО, паспорта, ИНН, счета, контакты). Собрал пайплайн маскирования PDF/DOCX полностью на opensource-стеке (pdfplumber, reportlab, pypdf, Tesseract) в условиях on-prem без доступа в интернет, что позволило **заменить ручное маскирование** документов и **разблокировать передачу документов внешним подрядчикам** (сотни документов в день).

Короткая версия (одним bullet'ом):

> Дообучил GliNER для маскирования ПДн в документах (Recall 0.90–0.95, F1 ≈ 0.90); собрал opensource-пайплайн redaction'а PDF/DOCX (pdfplumber + reportlab + pypdf + Tesseract) для on-prem без интернета; заменил ручной процесс, сотни документов/день.
