- **Рост маржи в A/B-тесте** по сравнению с контрольной группой SKU, управляемой по старой логике (точное число — TODO, заполнить реальным значением).
- **Объём продаж остался в рамках ограничения α** — модель не даёт «пожертвовать» объёмом ради маржи больше, чем согласовано с клиентом.
- **Реакция на акции и промо маркетплейсов** стала автоматической — модель видит промо-календарь и корректирует рекомендации заранее.
- **Объяснимость для категорийного менеджера.** Каждая рекомендация приходит с коротким текстом «почему именно эта цена», и менеджер может принять/отклонить её осознанно.

- **Модель эластичности:**
  - Sanity check: знак и порядок эластичности совпадают с ожиданиями по категориям.
  - Pooled-оценка по NLP-кластеру решает cold-start для новых SKU и «слепое пятно» на товарах с почти неизменной ценой.

  - Кластеризация описаний → стабильные семантические группы товаров, используемые в pooled-эластичности и cold-start.

## Формулировки для резюме

Длинная:

> **Спроектировал и реализовал** систему dynamic pricing для селлера на WB/Ozon в Webbee ML. Трёхчастная архитектура: **forecast спроса на LightGBM global model** по всем SKU с лаг-фичами и промо-календарём, **модель ценовой эластичности** по SKU и pooled по кластеру, **оптимизация цены** через argmax маржи с жёстким ограничением на объём в штуках. Добавил **NLP-подсистему**: кластеризацию товаров по описаниям на русскоязычных sentence-transformer'ах для cold-start новых SKU и pooled-эластичности, а также **product matching** с ценами конкурентов. Решил проблемы **out-of-stock** (маска спроса), **искажения эластичности промо-календарём маркетплейса** (исключение промо-периодов из обучения эластичности) и **малой вариации цены** (pooled-оценка по NLP-кластеру). Измерил эффект через **rolling-origin бэктест** (MAPE/MAE) и **A/B-тест по SKU** (uplift по марже при соблюдении ограничения на объём). Продакшн — ежедневный Airflow-DAG, публикация цен через API WB/Ozon, UI с ручным override для категорийного менеджера.

Короткая:
