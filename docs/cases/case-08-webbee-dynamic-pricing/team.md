# Команда и роли

**Состав:** 3 человека на проекте — два инженера со стороны **Webbee ML** и PM со стороны **клиента-селлера**.

## Состав

| Роль | Сторона | Зона ответственности |
|---|---|---|
| **Data Scientist (Артур)** | Webbee ML | Вся ML-часть: forecast спроса, модель эластичности, оптимизация цены, NLP-подсистема (кластеризация описаний + product matching), бэктест и A/B |
| **Backend-инженер** | Webbee ML | Сервис рекомендаций цен (FastAPI), витрины данных, Airflow-DAG'и, парсер цен конкурентов, интеграция с WB/Ozon API (выгрузка отчётов и публикация цен), Docker-деплой, UI-бэкенд для категорийного менеджера |
| **PM** | Клиент | Приоритеты, согласование списка SKU, бизнес-правил, MAP/RRP, приёмка, коммуникация с категорийными менеджерами внутри клиента |

**На стороне клиента** также присутствуют **категорийные менеджеры** — это не часть команды разработки, а пользователи системы; они принимают/отклоняют рекомендации и дают обратную связь.

## Моя зона ответственности (Артур, Data Scientist)

Как единственный DS на проекте, я закрывал ML-часть целиком:

### Data и фичи
- Совместно с backend'ом согласовал контракт витрин (продажи, цены, остатки, справочник товаров) и что конкретно пойдёт в feature store.
- Собрал **feature engineering**: лаги, окна, календарь, промо-флаги, отклонения цены от собственной и конкурентной, маски out-of-stock.

### Модели
- **Forecast спроса:** обучил и валидировал **LightGBM global model** по всем SKU, подобрал фичи, провёл rolling-origin backtest, сравнил с Prophet-baseline на стабильных SKU.
- **Модель эластичности:** построил регрессию `log(demand) ~ log(price)` с контролями, реализовал **исключение промо-периодов** и out-of-stock, реализовал **pooled-оценку по NLP-кластеру** для SKU с малой вариацией цены и новых товаров.
- **Оптимизация цены:** реализовал процедуру «дискретная сетка → forecast + корректировка эластичностью → argmax маржи с ограничением на объём», интегрировал слой бизнес-правил (MAP/RRP, стоп-листы, акции).

### NLP-подсистема
- Настроил **sentence-transformer** для русскоязычных названий и описаний товаров.
- Сделал **кластеризацию** товаров через HDBSCAN (внутри категорий).
- Построил **product matching** с товарами конкурентов (от парсера backend'а) через cosine similarity с порогом и с процессом ручной верификации топовых SKU.
- Интегрировал всё это в фичи: `cluster_id`, `pooled_elasticity`, `competitor_price_delta`.

### Валидация и A/B
- Реализовал **бэктест на истории**: реконструкция рекомендаций модели на прошлых периодах.
- Совместно с PM клиента **разработал дизайн A/B-теста по SKU** (стратификация по категориям и выручке, длительность, критерии значимости).
- Анализировал результаты бэктеста и A/B, считал **uplift по марже**.

### Объяснимость
- Реализовал **генерацию короткого текстового объяснения** для каждой рекомендации на основе ключевых факторов (forecast, эластичность, цена конкурентов, остатки, промо).
- Объяснения видит категорийный менеджер в UI перед тем, как принять или отклонить рекомендацию.

## Взаимодействие

- **С backend-инженером Webbee** — плотно, почти ежедневно: контракты витрин, формат артефактов модели, интеграция inference, отладка парсера конкурентов, формат ответа для UI.
- **С PM клиента** — на стартовых воркшопах (формализация бизнес-правил, α, MAP/RRP), при дизайне A/B, при приёмке и демо. Регулярные созвоны по результатам.
- **С категорийными менеджерами клиента** (через PM) — сбор обратной связи по конкретным рекомендациям: что кажется неправильным, где модель «тупит», на каких категориях больше всего override'ов.
