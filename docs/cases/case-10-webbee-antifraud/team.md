# Команда и роли

**Состав:** 2 инженера со стороны Webbee ML + антифрод-аналитик клиента как заказчик и источник разметки.

## Состав

| Роль | Сторона | Зона ответственности |
|---|---|---|
| **Data Scientist (Артур)** | Webbee ML | Вся ML-часть: supervised CatBoost/LightGBM, Isolation Forest, графовый анализ и Community Detection, фичи, валидация, метрики, SHAP-объяснения |
| **Backend-инженер** | Webbee ML | Realtime-сервис FastAPI + Redis-кэш графовых агрегатов, интеграция с чекаутом клиента, 3DS-интеграция с эквайером, очереди ручного разбора, хранилища, деплой, latency-оптимизации |
| **Антифрод-аналитик** | Клиент | Заказчик системы; источник ручной разметки по сомнительным транзакциям; пользователь UI (очередь ручного разбора с SHAP-объяснениями); обратная связь по ошибкам модели; коммуникация с эквайером по спорам |

## Моя зона ответственности (Артур, Data Scientist)

### Данные и фичи
- Собрал **обучающую выборку** из истории транзакций + chargeback'ов + вердиктов антифрод-аналитика.
- Построил **feature engineering**: поведенческие, транзакционные, аккаунтные, платёжные, сессионные, графовые. Всё — с учётом time-aware split'а, без утечек «из будущего».
- Продумал **заполнение отсутствующих фичей** для новых клиентов/устройств (cold-start в фрод-контексте).

### Supervised-модель
- Обучил и валидировал **CatBoost/LightGBM** на исторических парах «транзакция → chargeback / ручной_фрод».
- Подобрал **`scale_pos_weight`**, вёл эксперименты с undersampling, зафиксировал **PR-AUC** как главную метрику и **FPR-constraint** как критерий операционного порога.
- Проводил rolling-time валидацию с эмуляцией chargeback-задержки.

### Unsupervised-слой
- Обучил **Isolation Forest** на «нормальной» части распределения фичей.
- Настроил порог аномальности так, чтобы Isolation Forest **дополнял** supervised-модель: редкие сильные аномалии уходят отдельным флагом в агрегатор, даже если supervised их проморгал.
- Построил **канал «новых паттернов»:** аномалии, которые supervised не флагнул, — приоритетно в очередь аналитика.

### Графовый пайплайн
- Спроектировал **схему графа**: узлы-сущности (клиент/устройство/карта/адрес/email/IP), рёбра-совместные появления, веса по свежести.
- Реализовал построение графа через `networkx` в ночном Airflow-DAG'е.
- Запустил **Community Detection (Louvain)** и выделил для каждого узла ключевые агрегаты: `community_id`, размер, доля chargeback'ов в community за 30/60/90 дней, плотность связей.
- Спроектировал **schema Redis-кэша** графовых агрегатов вместе с backend-инженером.
- Добавил эти агрегаты как фичи в supervised-модель и в логику жёстких override'ов.

### Агрегатор и пороги
- Настроил **агрегацию** supervised-скора + unsupervised-скора + графовых флагов.
- Подобрал **пороги allow / 3DS / block** под целевой FPR, согласованный с клиентом.
- Явно заложил **мягкий исход `3DS challenge`** для серой зоны вместо жёстких блокировок — это снизило риск ложных блокировок честных клиентов.

### Объяснимость
- Настроил **SHAP-объяснения** для каждого решения модели.
- Реализовал **человеческую формулировку причин** по топ-фичам («mismatch billing/shipping, IP из high-risk сети, устройство в community с 7 chargeback'ами за 30 дней»).
- Лог решения с причинами сохраняется — критично для последующих споров с банком.

### Мониторинг и retrain
- Регулярный **retrain** на свежей истории chargeback'ов и вердиктов аналитика через Airflow.
- Настроил **drift-мониторинг** фичей и скоров через Evidently, с алертами при значимом сдвиге.
- Настроил **feedback loop:** каждый override'нутый аналитиком вердикт возвращается в обучающую выборку, приоритетно пересматривается на следующем retrain.

## Взаимодействие

- **С backend-инженером Webbee** — плотно на всех этапах: контракт API, структура Redis-кэша графовых агрегатов, latency-оптимизации, отладка production-инцидентов, 3DS-интеграция, UI очереди аналитика.
- **С антифрод-аналитиком клиента** — регулярно: сбор ручной разметки, разбор ошибок модели, обсуждение «новых паттернов» из Isolation Forest-канала, калибровка порогов под текущую ситуацию с фродом, демо новых фичей модели.
