# Кейс 10: Anti-fraud для D2C e-com — платёжный фрод и чарджбэки (Webbee ML)

## Краткое описание

**Работодатель:** Webbee ML — аутсорс ML-решений для e-commerce.

**Клиент:** **D2C-магазин** со своим чекаутом и платёжным процессингом через эквайера. Обезличено по NDA.

**Задача:** на этапе чекаута **в realtime** оценивать риск платёжной транзакции и принимать решение `allow / 3DS / block`, чтобы снижать чарджбэки и прямые убытки от фрода, не блокируя честных клиентов.

**Итог одной строкой:** многослойный пайплайн — **CatBoost/LightGBM** supervised-скор (обученный на истории чарджбэков и вердиктов аналитика) + **Isolation Forest** для новых паттернов + **графовый анализ** с **Community Detection** (Louvain) по графу «клиенты ↔ устройства ↔ карты ↔ адреса ↔ IP», фичи которого подмешиваются в supervised. Всё сходится в агрегатор и три исхода `allow / 3DS / block`. Graph + community считаются офлайн в batch'е, агрегаты лежат в **Redis**, inline-сервис укладывается в десятки миллисекунд SLA. **SHAP-объяснения** на каждое решение — для антифрод-аналитика клиента.
