# Достижения

## Бизнес-результаты

- **Снижение прямых убытков от chargeback'ов** — главный измеримый эффект системы. Точные цифры — TODO.
- **Снижение доли chargeback'ов** в общем потоке транзакций — ниже порога, за которым платёжные системы повышают комиссии / накладывают ограничения.
- **Честные клиенты не страдают:** FPR удерживается под согласованным с клиентом порогом, благодаря мягкому исходу `3DS challenge` для серой зоны вместо жёсткого `block`.
- **Снят ручной потолок** с антифрод-аналитика: раньше он вручную просматривал подозрительные транзакции и быстро перегружался; теперь он работает по **приоритизированной очереди** с объяснениями, а большая часть решений принимается автоматически.
- **Пойматы графовые кольца фрода,** которые правилами и одиночной supervised-моделью не ловились: мошеннические группы, расшаривающие карты/устройства/адреса между аккаунтами.

## Технические результаты

- **Многослойная архитектура:** supervised CatBoost/LightGBM + Isolation Forest + графовый анализ с Community Detection (Louvain). Каждый слой закрывает свою задачу и валидируется отдельно.
- **PR-AUC** по фроду на time-aware hold-out'е — главная техническая метрика (точное число — TODO). ROC-AUC приводится как вторичная.
- **Recall по фроду** при жёстком **ограничении FPR** — основной tradeoff, под который тюнились пороги (TODO: конкретная точка операционной работы).
- **Latency p99** realtime-сервиса укладывается в согласованный SLA в десятках миллисекунд (TODO: точное число), за счёт предвычисленных графовых фичей в Redis и параллельного инференса supervised + unsupervised.
- **Объяснимость:** каждое решение сопровождается **SHAP-объяснением** на естественном языке, лог решения сохраняется для последующих споров с банком.
- **Устойчивость к drift'у:** регулярный retrain + мониторинг через Evidently + канал «новых паттернов» через Isolation Forest.

## Формулировки для резюме

Длинная:

> **Спроектировал и реализовал** realtime anti-fraud систему для D2C e-commerce в Webbee ML. Многослойный пайплайн: **CatBoost/LightGBM** supervised-модель на исторических chargeback'ах и вердиктах аналитика + **Isolation Forest** для новых паттернов + **графовый анализ** с **Community Detection (Louvain)** по графу «клиенты ↔ устройства ↔ карты ↔ адреса ↔ IP», агрегированные графовые фичи (размер community, доля фрода в community за 30/60/90 дней, плотность связей) подмешиваются в supervised-модель. Работает **inline в чекауте** (десятки миллисекунд SLA) за счёт предвычисления графовых агрегатов офлайн и хранения в **Redis**, граф и community пересчитываются в ночном batch'е. Три исхода `allow / 3DS / block` с **SHAP-объяснениями** для антифрод-аналитика. Решил задачи **сильнейшего дисбаланса классов** (PR-AUC + `scale_pos_weight` + валидация в натуральной пропорции), **интерпретации для аналитика** (SHAP + текстовые причины + логи решений для споров с банком), **latency в realtime** (offline предвычисление графа + Redis), **адверсариального drift'а** (регулярный retrain + мониторинг через Evidently + канал «новых паттернов» через Isolation Forest).

Короткая:

> Realtime anti-fraud для D2C e-com: CatBoost + Isolation Forest + Community Detection по графу клиент/устройство/карта/IP с агрегатами в Redis; inline в чекауте, PR-AUC + FPR как главные метрики, SHAP-объяснения для аналитика, снижение $ chargeback'ов.
