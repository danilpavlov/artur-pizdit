## Зачем этот сайт

- Зафиксировать реальный опыт в структурированном виде.
- Научиться говорить о проектах на языке **бизнес-результата**, а не только моделей.
- Иметь готовый референс при заполнении резюме / LinkedIn / сопроводительных писем.

## Как это устроено

- **[Шаблон кейса](template/index.md)** — скелет, по которому описывается любой проект. Разделён на страницы: контекст, флоу, достижения, стек, команда, ретро.
- **[Кейсы](cases/index.md)** — конкретные проекты, заполненные по шаблону.
